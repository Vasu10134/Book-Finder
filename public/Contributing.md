## 1. Fork & Clone the Repository
```bash
# Fork the repository to your GitHub account
# Then clone it locally
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>

2. Create a New Branch

Use a meaningful branch name that describes your change:

git checkout -b feature/add-login-page


Examples of good branch names:

feature/improve-ui-sidebar

bugfix/fix-button-crash

docs/update-readme

3. Make Your Changes

Write clean, understandable code.

Follow the existing project structure and style.

Add comments if logic is complex.

4. Test Your Changes

Before submitting, ensure:

The app runs without errors

No functionality is broken

Code is formatted properly

If the project uses a formatter/linter, run it:

npm run format
npm run lint


(or equivalent commands if different)

5. Commit Your Changes

Use clear and descriptive commit messages:

git add .
git commit -m "Added sidebar navigation and improved UI layout"

Good Commit Message Examples:

docs: updated README with installation steps

fix: resolved crash when loading dashboard

feat: added search functionality for exams

6. Push & Submit a Pull Request
git push origin <your-branch-name>


Then open a Pull Request (PR) from your branch to main.

PR Guidelines:

Describe what you changed and why.

Attach screenshots if UI changes are made.

Reference related Issue number (if exists):

Closes #12

7. Code of Conduct

By contributing, you agree to follow our Code of Conduct:
See: CODE_OF_CONDUCT.md

8. Need Help?

If you're unsure about anything:

Open an Issue describing your doubt

Or ask directly in the discussion / chat channel (if available)
